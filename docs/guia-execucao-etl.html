O que essa pipeline faz?

A pipeline ETL do Elas++ é um programa Python que, toda vez que é executado, realiza três grandes tarefas automaticamente:

📥 Baixa a planilha
do SharePoint
→
🔄 Limpa e organiza
os dados
→
☁️ Salva no banco
PostgreSQL (Railway)

Após a execução, as tabelas do banco de dados ficam atualizadas e prontas para alimentar o dashboard de BI.

Pré-requisitos — instale uma vez só
1
Instalar Python 3.13 ou superior
O programa que vai rodar o nosso código

Acesse python.org/downloads e baixe a versão mais recente para o seu sistema. Durante a instalação no Windows, marque a opção "Add Python to PATH" antes de clicar em instalar.

Depois de instalar, verifique se funcionou abrindo o terminal e digitando:

Windows (PowerShell)
Mac / Linux
python --version
# Deve aparecer: Python 3.13.x
⚠️
No Mac/Linux, pode ser necessário usar python3 no lugar de python.
2
Instalar Git
Ferramenta para baixar e gerenciar o código

Acesse git-scm.com/downloads e instale a versão para o seu sistema operacional. Não é necessário alterar nenhuma opção padrão durante a instalação.

Verifique a instalação:

git --version
# Deve aparecer: git version 2.x.x
Baixar o código
3
Clonar o repositório
Baixar todos os arquivos do projeto para o seu computador

Abra o terminal (no Windows: pesquise por "PowerShell" no menu Iniciar) e execute:

git clone https://github.com/ICEI-PUC-Minas-PMV-SI/ellas.git
cd ellas

Isso vai criar uma pasta chamada ellas no seu computador com todo o código do projeto.

4
Criar o ambiente virtual
Um espaço isolado para instalar as dependências do projeto

O ambiente virtual evita conflitos com outros projetos Python no seu computador. Dentro da pasta ellas, execute:

Mac / Linux
python3 -m venv .venv
source .venv/bin/activate
Windows (PowerShell)
python -m venv .venv
.venv\Scripts\Activate.ps1

Quando ativado, você verá (.venv) no início da linha do terminal. Isso confirma que está dentro do ambiente virtual.

⚠️
Se no Windows aparecer uma mensagem de "execução de scripts desabilitada", execute este comando primeiro: 
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Configurar credenciais
5
Criar o arquivo de configuração .env
Onde ficam as senhas e endereços dos serviços

Na raiz da pasta ellas, crie um arquivo chamado exatamente .env (com o ponto na frente). Abra-o em qualquer editor de texto e cole o conteúdo abaixo, preenchendo cada valor com as informações fornecidas pelo responsável do projeto: 

# ── Banco de Dados PostgreSQL (Railway) ──────────────
POSTGRES_HOST=seu_host_aqui
POSTGRES_PORT=5432
POSTGRES_DB=railway
POSTGRES_USER=seu_usuario_aqui
POSTGRES_PASSWORD=sua_senha_aqui

# ── SharePoint (fonte dos dados) ──────────────────────
SHAREPOINT_URL=https://suaorganizacao.sharepoint.com/sites/ellas
SHAREPOINT_TENANT=id-do-tenant-azure
SHAREPOINT_CLIENT_ID=id-do-app-azure
SHAREPOINT_USER=seu_email@pucminas.br
SHAREPOINT_PASS=sua_senha
FILE_PATH=/sites/ellas/Documentos Compartilhados/planilha.xlsx
🔒
Nunca compartilhe o arquivo .env! Ele contém senhas e credenciais de acesso. Não o envie por e-mail, WhatsApp ou faça upload no GitHub. Esse arquivo já está configurado para ser ignorado pelo Git.
Referência das variáveis:
VariávelO que éDe onde obter
POSTGRES_HOST Endereço do banco de dados obrigatório Painel do Railway → seu banco → Connection Details
POSTGRES_PASSWORD Senha do banco obrigatório Painel do Railway → seu banco → Connection Details
SHAREPOINT_TENANT ID do tenant Azure obrigatório Responsável técnico do projeto / admin Azure AD
SHAREPOINT_CLIENT_ID ID do app registrado no Azure obrigatório Responsável técnico do projeto / admin Azure AD
SHAREPOINT_USER / PASS Login de acesso ao SharePoint obrigatório Suas credenciais institucionais PUC Minas
FILE_PATH Caminho da planilha no SharePoint obrigatório Responsável técnico do projeto
Instalar as dependências
6
Instalar os pacotes Python necessários
Feito uma única vez, ou quando o requirements.txt for atualizado

Com o ambiente virtual ativado (você verá (.venv) no terminal), execute: 

pip install -r requirements.txt

Isso vai instalar automaticamente todos os pacotes que o projeto precisa: pandas, SQLAlchemy, psycopg2 (driver do PostgreSQL), msal (autenticação SharePoint), entre outros. O processo pode demorar alguns minutos na primeira vez.

✅
Você saberá que funcionou quando aparecer: Successfully installed ... no terminal.
Executar a pipeline
7
Rodar o programa principal
Este é o comando que você vai usar toda vez que precisar atualizar os dados

Certifique-se de que:

  • ✓ Você está na pasta ellas
  • ✓ O ambiente virtual está ativado ((.venv) aparece no terminal)
  • ✓ O arquivo .env está criado e preenchido
  • ✓ Você tem acesso à internet

Então execute:

python src/main.py

O programa vai:

  • Conectar ao SharePoint e baixar a planilha mais recente
  • Processar e limpar os dados
  • Apagar os dados antigos do banco e inserir os novos
ℹ️
O processo demora cerca de 1 a 3 minutos dependendo da velocidade da sua internet e do tamanho da planilha. Não feche o terminal enquanto estiver rodando.
Verificar o resultado
8
Como saber se funcionou?
Sinais de sucesso e como interpretar mensagens de erro

Execução bem-sucedida — o terminal não exibirá erros e o programa encerrará normalmente. Em seguida, os dados já estarão disponíveis no banco PostgreSQL para o dashboard.

Erros comuns e como resolver:
❌
Falha na autenticação — "Falha na autenticação: ..."
O usuário ou senha do SharePoint estão errados no .env. Verifique as variáveis SHAREPOINT_USER e SHAREPOINT_PASS.
❌
Erro de conexão com o banco — "could not connect to server"
As credenciais do PostgreSQL estão incorretas ou o banco está offline. Verifique as variáveis POSTGRES_* no .env.
❌
Módulo não encontrado — "ModuleNotFoundError: No module named ..."
O ambiente virtual não está ativado, ou o pip install não foi executado. Volte ao Passo 4 e 6.
✅
Se tudo estiver correto e ainda assim der erro, entre em contato com o responsável técnico do projeto informando a mensagem de erro completa que apareceu no terminal.
Alternativa — usar Docker (avançado)
★
Executar via Docker
Sem precisar instalar Python — apenas Docker Desktop

Se você prefere não configurar o Python localmente, pode usar o Docker. Instale o Docker Desktop em docker.com/products/docker-desktop, depois execute dentro da pasta ellas: 

# 1. Construir a imagem (só precisa fazer uma vez)
docker build -t ellas-etl .

# 2. Executar passando o arquivo de credenciais
docker run --env-file .env ellas-etl
ℹ️
O arquivo .env ainda é necessário. Crie-o conforme o Passo 5 antes de rodar o Docker.
Resumo — sequência de comandos completa
# 1. Clonar o repositório
git clone https://github.com/ICEI-PUC-Minas-PMV-SI/ellas.git
cd ellas

# 2. Criar e ativar ambiente virtual
python3 -m venv .venv
source .venv/bin/activate        # Mac/Linux
# .venv\Scripts\Activate.ps1    # Windows

# 3. Instalar dependências
pip install -r requirements.txt

# 4. Criar o arquivo .env com as credenciais (veja Passo 5)

# 5. Executar a pipeline
python src/main.py
