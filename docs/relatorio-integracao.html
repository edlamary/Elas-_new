O projeto Elas++_Pessoal era um script único (download_sharepoint.py) responsável apenas por baixar a planilha do SharePoint e convertê-la para CSV. Na integração com o repositório principal (ellas), esse script foi absorvido e expandido em uma pipeline ETL estruturada com separação de responsabilidades, modelagem dimensional (Star Schema) e carga em banco de dados PostgreSQL hospedado em nuvem.

🔀 Mudanças Detalhadas por Componente

1. Ponto de entrada — main.py

  • novo
    Orquestrador de pipeline substituiu script linear
    O projeto pessoal não tinha main.py. O projeto integrado introduz um main() que instancia e encadeia todos os serviços via injeção de dependência, tornando cada etapa testável e substituível de forma independente.

2. Extração do SharePoint

  • refactor
    Script global → classe SharePointSpreadsheetSource
    As funções get_access_token() e download_xlsx_bytes() do script pessoal foram encapsuladas na classe SharePointSpreadsheetSource em src/services/sharepoint_spreadsheet_source.py. O método público passa a ser .fetch() → Path, retornando o arquivo temporário em vez de escrever CSV.
  • removido
    Exportação para CSV eliminada
    As funções xlsx_sheet_to_csv() e a escrita em output/formulario_feedback.csv foram removidas. Os dados agora fluem diretamente para o PostgreSQL pela pipeline.
  • novo
    Interface SpreadsheetSource adicionada
    Uma interface abstrata foi criada em src/services/interfaces/ para desacoplar a pipeline do SharePoint, permitindo trocar a fonte (ex.: arquivo local, S3) sem alterar o restante do código.

3. Transformação dos dados

  • novo
    SurveyExtractor — leitura de múltiplas abas
    Enquanto o script pessoal lia apenas uma aba específica (SHEET_NAME), o SurveyExtractor usa o ExcelReader para carregar todas as abas do arquivo e consolidar os dados de formulário e de escolas.
  • novo
    SurveyTransformer — limpeza e métricas
    Novo serviço responsável por: normalização de colunas, padronização de tipos, tratamento de valores ausentes, cálculo de métricas de engajamento e geração do dataframe grouped_programs_df para a tabela fato de resumo de eventos.
  • novo
    SchoolExtractor — fonte de dados de escolas
    Nova extração a partir da planilha de parceiros (Parceiros (escolas, entidades, etc).xlsx) para popular a dimensão de participantes com dados de escola de origem.

4. Carga — banco de dados PostgreSQL

  • novo
    SchemaMaker — carga no Star Schema
    O SchemaMaker recebe os DataFrames transformados e popula as 5 tabelas do Data Warehouse: dim_participante, dim_evento, dim_tempo, fato_atividades e fato_evento_resumo.
  • novo
    Classe Postgres — conexão via SQLAlchemy
    Criada em src/db/postgres.py, gerencia o pool de conexões para o banco Railway via SQLAlchemy. A pipeline inicia sempre com um TRUNCATE das tabelas para garantir idempotência nas cargas.

5. Dockerfile

  • alterado
    Python 3.12 → 3.13 e adição de dependências do sistema
    A imagem base foi atualizada. Foram adicionados gcc e libpq-dev via apt-get para compilar o driver nativo psycopg2 do PostgreSQL. O comando de entrada mudou de download_sharepoint.py para src/main.py.
